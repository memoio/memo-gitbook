
# 了解 x402 协议

### 什么是 x402？

[x402.org](https://x402.org/) 是一个开放的支付协议，将稳定币支付能力直接带入纯粹的 HTTP 交互之中。

它重新启用了 `HTTP 402 Payment Required` 状态码，让服务器可以无缝为 API 或数字内容收取费用。

客户端（人类用户或 AI 代理）无需注册账号、无需 API Key、也无需复杂鉴权流程，即可通过程序化方式完成支付并获取资源。

### 为什么是 x402？

传统支付通道并不是为 Web 的速度或机器对机器（M2M）交互场景设计的：

- 它们缓慢、昂贵，并且依赖注册与密钥配置。

x402 将“支付”直接嵌入到 Web 原生的请求–响应流中，使得即时、全球化、按使用计费的支付模式可以低成本集成——同时适用于人类用户与自治智能体。

### 优势

- 基于 402 状态码的简单 HTTP 集成
- 按请求/按使用量付费的定价模式
- 使用稳定币进行微支付（例如：Solana 上的 USDC）
- 原生适配智能代理：AI Agent 可自动发现与完成支付
- 零摩擦：无需账号、无需 API Key、无需会话管理
